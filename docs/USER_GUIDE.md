# 🚀 Autensa (Mission Control) 用户使用全指南

Autensa (Mission Control) 是一个为 AI 代理集群设计的“指挥中心”。它不仅是任务看板，更是协调多代理协作、管理复杂工作流的智能引擎。

---

## 1. 核心架构：指挥体系

在使用前，请理解以下核心实体：
*   **工作区 (Workspace)**：独立的项目环境，拥有专属的 Slug（如 `website-refactor`）和成员。
*   **智能代理 (Agent)**：通过 OpenClaw Gateway 接入的执行实体。
*   **工作流 (Workflow)**：定义任务从创建到完成的一系列阶段（Stages），如 `PLANNING → IN_PROGRESS → TESTING → DONE`。

---

## 2. 智能代理 (Agent) 深度配置

代理的效能取决于其“人设”与“规则”的精细程度。在 `Agent Modal` 中，你可以通过四个维度定义代理：

### A. 基础信息 (Info)
*   **角色 (Role)**：关键属性。工作流引擎通过匹配此字段（如 `Developer` 或 `Reviewer`）自动分配任务。
*   **模型 (Model)**：选择底座 LLM。建议为复杂逻辑分配 `GPT-4/Claude-3`，为简单审查分配轻量模型。
*   **主助手 (Master Agent)**：每个工作区应有一个主助手，负责整体规划和子任务派发。

### B. 灵魂 (Soul.md) - **核心人设**
这是代理的系统提示词（System Prompt）。
*   **使命定义**：明确代理的目标（例如：“你是一个严谨的代码审查员，专注于安全性和性能改进”）。
*   **性格偏好**：定义沟通语气、价值观及决策风格。
*   **技能定义**：虽然“技能”是底层工具的体现，但在此处描述“你擅长使用 X 工具处理 Y 任务”能显著提高模型调用工具的意图准确率。

### C. 用户交互准则 (User.md)
定义代理如何与人类协作。
*   例如：“在执行破坏性操作前必须询问用户”、“汇报进度时请使用简洁的 Markdown 表格”。

### D. 团队协作准则 (Agents.md)
定义代理在多助手环境下的行为。
*   例如：“如果你发现代码逻辑错误，请将任务退回给 `Developer` 角色并附带详细日志”。

## 3. 工作流与自动调度 (Workflow Engine)

Autensa 的核心价值在于 **自动流转**。

### 角色自动映射
当任务进入新阶段时，引擎会扫描当前工作区：
1.  寻找 `Role` 字段与阶段要求完全匹配的代理。
2.  若无精确匹配，则寻找名称或描述中包含该角色的代理。
3.  自动将 `assigned_agent_id` 切换为该代理，并触发 `Dispatch`（派发）。

### 失效回溯 (Fail-Loopback)
在 `Testing` 或 `Review` 阶段，如果代理判定任务不合格（调用 `fail` 接口）：
*   引擎会根据工作流配置的 `fail_targets`，自动将任务回滚到指定阶段（如回滚到 `IN_PROGRESS`）。
*   任务会自动重新分配给之前的执行者，并带上失效原因。

### 队列机制 (Queue Stages)
某些阶段（如 `Verification`）可能没有指定代理。此时它们作为队列存在。
*   引擎会自动“排干”队列：当后续阶段空闲时，自动将队列中最旧的任务推向下一环节。

---

## 4. 任务规划与规范 (Planning & Spec)

不要直接运行复杂的任务，先利用 **规划 (Planning)** 功能：
1.  **AI 提问**：代理会在 `PLANNING` 阶段向你提出一系列问题（分类包括：目标、范围、技术栈等）。
2.  **锁定规范 (Spec)**：回答完毕后，系统会生成一份 Markdown 格式的执行规范。
3.  **确认派发**：一旦 Spec 被锁定，任务才会正式进入执行流。这确保了 AI 充分理解了你的意图。

---

## 5. 实时监控与调试

### Live Feed (实时动态)
通过看板右侧的实时流，你可以看到：
*   代理产生的每一步 **Activity**（如“正在分析文件结构”、“正在编写单元测试”）。
*   实时产出的 **Deliverables**（交付物）：代码片段、生成的图片、文档链接等。

### SSE 调试面板
如果发现页面数据没有自动更新，请检查右下角的 **SSE Panel**。它显示了与后端的实时连接状态。

---

## 6. 高级技巧 (Pro Tips)

*   **技能扩展**：通过在 OpenClaw Gateway 端注册新的 `Tools`，并在 Autensa 代理的 `Soul.md` 中描述这些工具的使用方法，实现能力的无限扩展。
*   **自定义工作流**：在数据库中配置自定义 `workflow_templates`，可以为不同性质的项目（如“文档翻译” vs “代码重构”）定制完全不同的流转逻辑。
*   **交付物追溯**：所有代理产生的产物都会记录在 `Task Deliverables` 中，你可以随时点击预览或下载，即便任务已经标记为 `DONE`。
